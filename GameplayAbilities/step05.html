STEP 05

GameplayEffect でダメージを与える

UE 5.5 対応

GE_Damage を Blueprint で作り、アビリティから適用して Health を減らす。

このステップのゴール

STEP 04 で用意した Health を、実際に変化させます。 GAS では Attribute の値を直接書き換えるのではなく、 GameplayEffect（以下 GE） という「命令書」を経由して変化させます。

  • GameplayEffect の仕組みを理解する
  • ブループリントで GE_Damage を作る（Health を 20 減らす）
  • GA_MyFirstAbility を更新してダメージを適用する
  • Q を押すたびに Health が 20 減ることを確認する

このステップはブループリントのみで完結します。
C++ を一切書かずに「ダメージを与える」という処理が作れます。

GameplayEffect とは

GameplayEffect は「Attribute をどう変化させるか」を定義したデータアセットです。 アビリティや他のゲームコードから GE を適用することで、 HP を減らしたり、バフをかけたり、クールダウンを管理したりできます。

GE には3種類の Duration Policy（持続時間の種類）があります。

Duration Policy 動作 主な用途
Instant 適用した瞬間に値が変わる ダメージ、即時回復、即時消費
Duration 指定した秒数だけ効果が続く 毒ダメージ（DoT）、一時的なバフ
Infinite 明示的に除去するまで効果が続く 装備ボーナス、パッシブバフ

今回は最もシンプルな Instant を使います。

GE_Damage を作る

1
GameplayEffect ブループリントを作成する

コンテンツブラウザを右クリックして 「ブループリントクラス」 を選択します。 親クラスの検索欄に GameplayEffect と入力して選択し、 名前を GE_Damage にしてください。

2
Duration Policy を Instant に設定する

GE_Damage を開き、「クラスのデフォルト」 （ブループリントエディタ上部のボタン）を選択します。 詳細パネルに GE の設定が表示されます。

「Gameplay Effect → Duration Policy」 を探して、 Instant に設定してください。

3
Modifier を追加する

詳細パネルの 「Gameplay Effect → Modifiers」 セクションの 「+」ボタンを押して Modifier を1つ追加します。 以下のように設定してください。

Modifier の設定値
Attribute              : MyAttributeSet.Health
Modifier Op            : Add
Modifier Magnitude
  Magnitude Calculation Type : Scalable Float
  Scalable Float Magnitude   : -20
「Add」で -20 を指定する理由

GAS の Modifier Operation に「Subtract」はありません。 減算したい場合は Add に負の値（-20）を指定します。

Scalable Float とは

アビリティのレベルに応じて値をスケーリングできる仕組みです。 今は固定値なのでシンプルに -20 と入れるだけで動きます。 レベルシステムと連携させる場合は DataTable でカーブを定義できます。

設定が終わったらブループリントをコンパイルして保存してください。

GA_MyFirstAbility を更新する

STEP 03 で作った GA_MyFirstAbility を開き、 ActivateAbility のグラフを以下のように更新します。

4
ActivateAbility に GE_Damage の適用を追加する

既存の Print String の代わりに、以下の流れに組み替えてください。

Event ActivateAbility → Commit Ability
Commit Ability の Return Value を Branch につなぐ ↓

└ true → Apply Gameplay Effect to Owner
Gameplay Effect Class: GE_Damage → Get Ability System Component
Target: Get Owner Actor → Get Numeric Attribute Value
Attribute: MyAttributeSet.Health → Print String → End Ability
└ false → End Ability (Was Cancelled: true)

Apply Gameplay Effect to Owner は GameplayAbility ブループリントの中から 右クリックで検索すると見つかります。 Gameplay Effect Class に GE_Damage を設定してください。

GE を適用した後、現在の Health 値を確認するために Get Numeric Attribute Value → Print String をつなぎます。 float を文字列に変換する ToString (float) ノードが自動で挿入されます。

Apply Gameplay Effect to Owner が見つからない場合

GameplayAbility の外（Character Blueprint など）からは Apply Gameplay Effect to Owner は使えません。キャラクターの Blueprint から操作したい場合は Get Ability System Component → Apply Gameplay Effect to Target を使います（Target に自分自身を渡す）。

ブループリントをコンパイルして保存してください。

動作確認

5
プレイして Health が減ることを確認する

プレイを開始し、Q キーを押すたびに 画面左上に表示される数値が減っていけば成功です。

初期値 100 から始まり、 1回押すごとに 80 → 60 → 40 → 20 → 0 と変化するはずです。

0 以下になったときの挙動

GAS はデフォルトでは Health が 0 以下になることを許可しています（マイナスにもなります）。 「0 未満にならない」制御は AttributeSet の PostGameplayEffectExecute 関数でクランプ処理を追加するのが一般的ですが、これは応用的な内容なため今回は省略します。

うまくいかないときの確認ポイント

  • GE_Damage の Duration Policy が Instant になっているか
  • Modifier の Attribute が MyAttributeSet.Health になっているか（MaxHealth ではなく）
  • Modifier の Scalable Float Magnitude が -20（マイナス）になっているか
  • Apply Gameplay Effect to Owner に GE_Damage が設定されているか
  • Health の表示が変化しない → End Ability が呼ばれているか、再発動できているか確認

このステップのまとめ

「アビリティ → GameplayEffect → Attribute の変化」という GAS の基本的な流れが完成しました。
直接値を書き換えるのではなく GE を経由することで、 後で学ぶ GameplayTag による制御や バフの重ね掛け管理が自然に機能するようになります。

次のステップでは GameplayTag を学びます。 「スタン中はアビリティが使えない」といった状態管理を、 コードを書かずにタグの設定だけで実現できます。
