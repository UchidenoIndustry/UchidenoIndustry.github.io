STEP 04

AttributeSet の基本

UE 5.5 版

C++ で HP などのステータスを定義し、GAS から読み書きできる状態にする。

このステップのゴール

アビリティで「ダメージを与える」「回復する」といった処理を作るには、 まず「HP」や「MP」といったステータスを GAS に認識させる仕組みが必要です。 それが AttributeSet です。

  • AttributeSet とは何かを理解する
  • C++ で UMyAttributeSet クラスを作る（MaxHealth、Health を定義）
  • キャラクターに AttributeSet を追加する
  • ブループリントから HP の値を読めることを確認する

AttributeSet の定義は C++ が必須です。
ブループリントだけでは作れません。ただしコードは決まったパターンのコピペなので、 C++ に慣れていなくても大丈夫です。一度書いてしまえば、後はブループリントで操作できます。

AttributeSet とは

GAS における AttributeSet は「ステータスの入れ物」です。 キャラクターの HP、MP、攻撃力、防御力など、変化する数値をここにまとめて定義します。

普通の変数（float HPなど）との違いは何でしょうか？

普通の変数 AttributeSet の Attribute
値の変更 直接代入 GameplayEffect 経由（後のステップで説明）
変化の検知 自前で監視が必要 GAS が自動的に通知（OnAttributeChanged）
ネットワーク同期 自前でレプリケーション設定が必要 GAS が管理（後のステップで対応）
バフ・デバフ ロジックを自前で組む必要がある GameplayEffect で宣言的に適用できる

Attribute を GAS に管理させることで、後のステップで学ぶ 「GameplayEffect でダメージを与える」「クールダウン中は使えない」 といった仕組みが自然に使えるようになります。

AttributeSet の C++ クラスを作る

1
新規 C++ クラスを作成する

UE エディタのメニューから 「ツール → 新規 C++ クラス」 を開きます。

親クラスの選択画面で 「すべてのクラス」 にチェックを入れ、 検索欄に AttributeSet と入力します。 AttributeSet（GameplayAbilities モジュール）が見つかったら選択してください。

クラス名を MyAttributeSet にして作成します。 Visual Studio または Rider が自動的に開きます。

2
ヘッダーファイルを編集する

生成された MyAttributeSet.h を以下のように書き換えてください。 ファイルの内容をすべて置き換えます（既存コードは削除）。

MyAttributeSet.h
#pragma once

#include "CoreMinimal.h"
#include "AttributeSet.h"
#include "AbilitySystemComponent.h"
#include "MyAttributeSet.generated.h"

// Attribute のゲッター・セッターを自動生成する便利マクロ
#define ATTRIBUTE_ACCESSORS(ClassName, PropertyName) \
    GAMEPLAYATTRIBUTE_PROPERTY_GETTER(ClassName, PropertyName) \
    GAMEPLAYATTRIBUTE_VALUE_GETTER(PropertyName) \
    GAMEPLAYATTRIBUTE_VALUE_SETTER(PropertyName) \
    GAMEPLAYATTRIBUTE_VALUE_INITTER(PropertyName)

UCLASS()
class UMyAttributeSet : public UAttributeSet
{
    GENERATED_BODY()

public:
    UMyAttributeSet();

    // HP の最大値
    UPROPERTY(BlueprintReadOnly, Category = "Attributes")
    FGameplayAttributeData MaxHealth;
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, MaxHealth)

    // HP の現在値
    UPROPERTY(BlueprintReadOnly, Category = "Attributes")
    FGameplayAttributeData Health;
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, Health)
};
FGameplayAttributeData とは

ただの float ではなく、GAS 専用の数値型です。 「基本値（BaseValue）」と「現在値（CurrentValue）」の2つを内部で持っており、 バフ・デバフによる一時的な変化を管理できます。

ATTRIBUTE_ACCESSORS マクロについて

このマクロは GAS の標準ヘッダーにある4つのマクロを1行にまとめたものです。 GetHealth()・SetHealth()・InitHealth() などの関数を自動生成します。 中身を理解しなくてもコピペで動くので、今は「おまじない」として扱ってください。

3
cpp ファイルを編集する

MyAttributeSet.cpp を以下のように書き換えてください。 コンストラクタで HP の初期値（100）を設定しています。

MyAttributeSet.cpp
#include "MyAttributeSet.h"

UMyAttributeSet::UMyAttributeSet()
{
    InitMaxHealth(100.f);
    InitHealth(100.f);
}
InitHealth() と SetHealth() の違い

InitXxx() はゲーム開始時の初期化専用で、BaseValue と CurrentValue の両方を同時に設定します。 SetXxx() は CurrentValue だけを変更します。 コンストラクタでの初期値設定には InitXxx() を使うのが正しいパターンです。

キャラクターに AttributeSet を追加する

AttributeSet はキャラクターのコンポーネントとして追加します。 AbilitySystemComponent が同じ Actor 上の AttributeSet を自動的に認識します。

4
Character.h を編集する

[プロジェクト名]Character.h に 緑色の行を追加してください。

[プロジェクト名]Character.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Character.h"
#include "AbilitySystemInterface.h"
#include "MyAttributeSet.h"  // 追加
#include "[プロジェクト名]Character.generated.h"

class UAbilitySystemComponent;

UCLASS()
class A[プロジェクト名]Character : public ACharacter, public IAbilitySystemInterface
{
    GENERATED_BODY()

public:
    A[プロジェクト名]Character();
    virtual UAbilitySystemComponent* GetAbilitySystemComponent() const override;

protected:
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "GAS")
    TObjectPtr<UAbilitySystemComponent> AbilitySystemComponent;

    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "GAS")
    TObjectPtr<UMyAttributeSet> AttributeSet;  // 追加

    // 既存のコードはそのまま...
};
5
Character.cpp を編集する

[プロジェクト名]Character.cpp のコンストラクタに 緑色の行を追加してください。

[プロジェクト名]Character.cpp（コンストラクタ部分）
A[プロジェクト名]Character::A[プロジェクト名]Character()
{
    // 既存のコードはそのまま...

    AbilitySystemComponent = CreateDefaultSubobject<UAbilitySystemComponent>(TEXT("AbilitySystemComponent"));
    AttributeSet = CreateDefaultSubobject<UMyAttributeSet>(TEXT("AttributeSet"));  // 追加
}
なぜ ASC に直接登録しないのか

CreateDefaultSubobject で Actor のコンポーネントとして作成すると、 AbilitySystemComponent が同じ Actor 上の AttributeSet を自動的に検出して管理下に置きます。 明示的に「この ASC にこの AttributeSet を渡す」という操作は不要です。

6
コンパイルする

Visual Studio または UE エディタの「コンパイル」ボタンでビルドします。 エラーがなければ次へ進んでください。

コンパイルが通ったら、キャラクターを選択して詳細パネルを確認すると、 「GAS」カテゴリに AttributeSet が追加されているはずです。

ブループリントから Health の値を読む

AttributeSet が正しく追加されているか、ブループリントから値を読んで確認します。

7
BeginPlay で Health を表示する

キャラクターのブループリント（BP_ThirdPersonCharacter など）を開き、 Event BeginPlay に以下のノードをつないでください。

Event BeginPlay → Get Ability System Component → Get Numeric Attribute Value
Attribute: MyAttributeSet.Health → Print String
In String: (float を接続)

Get Numeric Attribute Value ノードの Attribute ドロップダウンから MyAttributeSet.Health を選択してください。

戻り値の Return Value（float）を Print String の In String に接続します。 型が異なるため、間に ToString (float) ノードが自動挿入されます。

Attribute がドロップダウンに出てこない場合

一度コンパイルして保存し、エディタを再起動してみてください。 コンパイル直後は反映されないことがあります。

8
プレイして確認する

プレイボタンを押すと、画面左上に 100.0 と表示されれば成功です。 MyAttributeSet.cpp のコンストラクタで設定した初期値が 正しく読み取れていることが確認できます。

うまくいかないときの確認ポイント

  • MyAttributeSet.h に #include "AbilitySystemComponent.h" があるか（ATTRIBUTE_ACCESSORS マクロに必要）
  • Character.h で #include "MyAttributeSet.h" を追加したか
  • コンストラクタの CreateDefaultSubobject<UMyAttributeSet> を追加したか
  • ブループリントの Get Numeric Attribute Value で Attribute に MyAttributeSet.Health を選んでいるか
  • Attribute がドロップダウンに表示されない → エディタを再起動してみる

このステップのまとめ

AttributeSet により、HP などのステータスが GAS の管理下に入りました。
定義は C++、読み書きはブループリントというハイブリッドな使い方が GAS のスタンダードです。

次のステップでは GameplayEffect を使って この Health を実際に減らす（ダメージを与える）処理を作ります。 ここからはほぼブループリントだけで進められます。
