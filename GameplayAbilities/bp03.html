BP STEP 03

最初の Ability を作る

UE 5.5 対応

ブループリントで GameplayAbility を作り、キャラクターに持たせてキーで発動させる。

このステップのゴール

前のステップで GAS を使える状態になりました。 このステップではいよいよ「アビリティ」を作って動かします。 難しいことは後回しにして、まず「発動する」ところまで進めましょう。

  • GameplayAbility ブループリントを作る
  • 発動時にメッセージを表示する（動作確認のため）
  • キャラクターにアビリティを持たせる
  • キーを押したらアビリティが発動する

GameplayAbility の基本構造

GameplayAbility ブループリントには、必ず意識しておくべき3つの処理があります。

①
ActivateAbility — アビリティ発動時に呼ばれる

アビリティが発動したときに最初に呼ばれる処理です。 「ジャンプする」「炎を出す」「回復する」など、アビリティの本体はここに書きます。

②
CommitAbility — コストとクールダウンを消費する

「MPを消費する」「クールダウンを開始する」といった処理を確定させます。 コストやクールダウンを設定していない場合でも、呼ぶ必要があります。 CommitAbility が false を返した場合（コスト不足など）は アビリティをキャンセルしてください。

③
EndAbility — アビリティを終了する

アビリティの処理が終わったら必ず呼びます。 これを呼ばないと GAS がアビリティを「まだ実行中」と認識し続けるため、 同じアビリティを再度発動できなくなったり、クールダウンが始まらなかったりします。

まとめると

ActivateAbility → やりたい処理 → CommitAbility → EndAbility、という流れが基本です。

Ability ブループリントを作る

1
ブループリントクラスを作成する

コンテンツブラウザの空白部分を右クリックして 「ブループリントクラス」 を選択します。

親クラスの選択画面が開くので、検索欄に GameplayAbility と入力して選択し、 ブループリントの名前を GA_MyFirstAbility にしてください。

命名のルール（慣例）

GAS 関連のアセットは接頭辞で種類がわかるように命名するのが一般的です。GameplayAbility は GA_ のように付けます。

2
ActivateAbility をオーバーライドする

GA_MyFirstAbility をダブルクリックして開きます。 ブループリントエディタ上部のメニューから 「オーバーライド → ActivateAbility」 を選択してください。

Event ActivateAbility ノードがグラフに追加されます。 このノードから処理をつないでいきます。

3
発動時の処理を組む

以下のようにノードをつないでください。 右クリックでノードを検索・追加できます。

Event ActivateAbility → Print String
In String: "Ability Activated!" → Commit Ability
Ability Handle: (自動接続)
Commit Ability の Return Value（bool）を分岐させる ↓
└ true → End Ability
Was Cancelled: false
└ false → End Ability
Was Cancelled: true

Commit Ability ノードには Ability Handle と Activation Info という入力があります。 Event ActivateAbility の出力ピンから引っ張ると自動的に接続できます。

Commit Ability の戻り値（Return Value）を Branch ノード（if 文）につなぎ、 True のときは通常終了、False のときはキャンセル終了としてください。

今の段階では Commit が false になることはない

コストもクールダウンも設定していないので、Commit Ability は必ず true を返します。ただし「失敗したらキャンセル」のパターンを最初から習慣にしておくと、後で困りません。

ブループリントをコンパイルして保存してください。

キャラクターに Ability を持たせる

GAS では「キャラクターが持っているアビリティの一覧」を AbilitySystemComponent が管理しています。 アビリティを発動するには、まず GiveAbility で キャラクターにアビリティを登録する必要があります。

4
キャラクターの Blueprint を開く

BP STEP 02 で AbilitySystemComponent を追加したキャラクターのブループリントを開いてください。 サードパーソンテンプレートの場合は Content/ThirdPerson/Blueprints/BP_ThirdPersonCharacter です。

5
BeginPlay で GiveAbility を呼ぶ

イベントグラフの Event BeginPlay を探し（なければ右クリックで追加）、 以下のようにノードをつないでください。

Event BeginPlay → Get Ability System Component → Give Ability
Ability Class: GA_MyFirstAbility

Get Ability System Component は、BP STEP 02 で実装した AbilitySystemInterface 経由で ASC を取得するノードです。 Target には Self（このキャラクター自身）を接続してください。

Give Ability の Ability Class に GA_MyFirstAbility を設定します。

Give Ability の戻り値について

Give Ability は FGameplayAbilitySpecHandle という値を返します。これは「このアビリティを識別するID」で、後で特定のアビリティを取り消したいときに使います。今は無視してかまいません。

キーを押して Ability を発動する

6
キー入力でアビリティを発動する

同じキャラクターのブループリントで、キーボードイベントを追加します。 グラフの空白部分を右クリックして Q キーのイベントを追加してください （Keyboard Events → Q）。

Q（Pressed） → Get Ability System Component → Try Activate Ability By Class
Ability Class: GA_MyFirstAbility

Try Activate Ability By Class は、指定したクラスのアビリティを 発動しようとするノードです。 発動の条件（コスト・クールダウンなど）を満たしていれば発動し、 満たしていなければ何もしません。

ブループリントをコンパイルして保存してください。

動作確認

7
プレイして確認する

エディタ上部の 「プレイ」ボタン を押してゲームを開始します。 Q キーを押すと、画面左上に "Ability Activated!" というメッセージが表示されれば成功です。

Q を押すたびにメッセージが出ることを確認してください。

うまくいかないときの確認ポイント

  • キャラクターの Blueprint に AbilitySystemComponent が追加されているか（BP STEP 02 の手順）
  • BeginPlay の Give Ability で GA_MyFirstAbility が正しく設定されているか
  • GA_MyFirstAbility 内で End Ability を呼んでいるか（呼ばないと2回目以降発動しない）
  • Get Ability System Component の Target に Self が接続されているか

このステップのまとめ

GameplayAbility の基本的な流れが完成しました。
「GiveAbility で登録 → TryActivateAbilityByClass で発動」 という2ステップが GAS の基本パターンです。

現在のアビリティは Print String するだけですが、 この枠組みの中にどんな処理でも入れられます。 次のステップでは GameplayTag を使って 「スタン中はアビリティを発動できない」という制御を実装します。
