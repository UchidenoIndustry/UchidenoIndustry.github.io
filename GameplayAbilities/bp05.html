BP STEP 05

攻撃アビリティを作る

UE 5.5 版

Play Montage and Wait でアニメーションと同期した攻撃アビリティを作る。発動中は再発動できない。

このステップのゴール

これまでの GA_MyFirstAbility は Print String を表示するだけでした。 このステップでは Play Montage and Wait を使い、 アビリティの発動中にアニメーションを再生します。 アニメーションが終わると自動でアビリティも終了し、 途中でも再発動できないよう制御します。

  • 攻撃アニメーションから AM_Attack（Animation Montage）を作成する
  • GA_Attack Blueprint を作り、Play Montage and Wait でモンタージュを再生する
  • Block Abilities with Tag で発動中の再発動を防ぐ
  • Q キーで攻撃アニメーションが再生され、完了まで再発動できないことを確認する

Play Montage and Wait は Ability Task の一種です。 アビリティがモンタージュの再生中「一時停止」し、 アニメーションが完了・中断されたときに続きを実行します。 Tick やタイマーを書かなくても、GAS がアニメーションの長さに合わせて自動で管理します。

Animation Montage を用意する

1
攻撃アニメーションと AM_Attack を準備する

Third Person Template には攻撃アニメーションが含まれていないため、 以下のいずれかの方法でアセットを用意してください。

アニメーションの入手方法（いずれか 1 つ）

Marketplace の Animation Starter Pack（無料）: Epic Games Launcher のライブラリから「Animation Starter Pack」をプロジェクトに追加できます。 UE5 Manny への IK Retarget が必要です。

Mixamo（Adobe アカウントがあれば無料）: mixamo.com から攻撃アニメーションを FBX でダウンロードしてインポートできます。

既存アニメーションを代替使用: MM_Land など Third Person Template のアニメーションをそのまま使うと、 見た目は攻撃に見えませんが「GAS とモンタージュの連携」の動作確認はできます。

アニメーションアセットが用意できたら、Content Browser（コンテンツブラウザ）でそのアセットを 右クリック → Create（作成） → Animation Montage（アニメーションモンタージュ） を選択します。名前を AM_Attack にします。

AM_Attack を開き、Slot が 「DefaultGroup.DefaultSlot」 になっていることを確認してください。

モンタージュが視覚的に再生されない場合（AnimBP の Slot 設定）

キャラクターの Animation Blueprint（ABP_Manny など）の AnimGraph に Slot ノード（名前: DefaultSlot） が必要です。 AnimGraph を開いて Output Pose の手前に挿入してください（右クリック → 「Slot」で検索）。 この設定がないとアニメーションは視覚的に再生されませんが、 GAS のイベント処理自体は正常に動作します。

GA_Attack を作成する

2
GA_Attack Blueprint を作成する

Content Browser で右クリック → Blueprint Class（ブループリントクラス） を選択。 親クラスの検索欄に 「GameplayAbility」 と入力して選択し、 名前を GA_Attack にして作成します。

GA_Attack を開くと、イベントグラフに Event ActivateAbility がすでに配置されています。

アビリティのグラフを組む

3
Play Montage and Wait でモンタージュを再生する

イベントグラフの空白を右クリックして 「Commit Ability」 を検索して配置し、 Event ActivateAbility の実行ピンに繋ぎます。

続けて 「Play Montage and Wait」 を検索して配置します。 Montage to Play に AM_Attack を設定します。

Play Montage and Wait の出力ピンを以下のように繋ぎます。 いずれも End Ability に繋いでください。

GA_Attack のグラフ
Event ActivateAbility → Commit Ability → Play Montage and Wait
Montage to Play: AM_Attack
[On Completed] → End Ability
[On Blend Out] → End Ability
[On Interrupted] → End Ability
[On Cancelled] → End Ability
4 本の出力ピンをすべて End Ability に繋ぐ。
OnCompleted と OnBlendOut の違い

On Completed はモンタージュが最後のフレームまで再生された瞬間、 On Blend Out はその直前の終了フェード開始時に発火します。 どちらも End Ability に繋ぐのが一般的で、2 回目の End Ability は内部で無視されます。

ブループリントをコンパイルして保存してください。

Block Abilities with Tag を設定する

このままでは攻撃モーション中に Q を連打すると攻撃が重なってしまいます。 Block Abilities with Tag を設定して、 発動中は同じアビリティが再発動できないようにします。

4
Ability.Attack タグを登録して GA_Attack に設定する

GA_Attack の Class Defaults（クラスのデフォルト）を開き、 Tags セクション（詳細パネルで「Tags」と検索）の AssetTags (Default AbilityTags) の「+」をクリックします。 タグブラウザで Ability.Attack を新規作成して設定します（BP STEP 04 の手順と同じです）。

同じく Block Abilities with Tag にも Ability.Attack を設定します。

GA_Attack の Tags 設定まとめ
AssetTags (Default AbilityTags) : Ability.Attack
   → このアビリティ自身のタグ

Block Abilities with Tag         : Ability.Attack
   → 発動中は Ability.Attack タグを持つアビリティをブロックする
   → 自分自身の AssetTag と一致するため、発動中は自分の再発動がブロックされる
Block Abilities with Tag の動作

「この GA_Attack が発動している間、Ability.Attack タグを持つアビリティをブロックする」という設定です。 GA_Attack の AssetTags に Ability.Attack があるため、 モンタージュが終わって End Ability が呼ばれるまで自分自身も発動できなくなります。

ブループリントをコンパイルして保存してください。

Q キーに割り当てる

5
BeginPlay で Give Ability、Q キーで TryActivate を設定する

BP_ThirdPersonCharacter のイベントグラフを開き、 Event BeginPlay に Give Ability（Ability Class: GA_Attack） を追加します。

Q キーの入力イベントに Try Activate Ability By Class（In Ability to Activate: GA_Attack） を繋ぎます。

以前の GA_MyFirstAbility について

BP STEP 03 で作った GA_MyFirstAbility の Give Ability・Q キーの TryActivate は 削除するか、別のキーに変更してください。

ブループリントをコンパイルして保存してください。

動作確認

6
攻撃アニメーションと再発動防止を確認する
  • Q キーを押す → 攻撃アニメーションが再生される
  • アニメーション中に Q を連打 → アニメーション完了まで新しい攻撃が始まらない
  • アニメーション完了後に Q → 再び攻撃アニメーションが再生される
アニメーションが視覚的に再生されない場合

AnimBP（ABP_Manny など）の AnimGraph に「DefaultSlot」という名前の Slot ノードが Output Pose の手前に挿入されているか確認してください（Step 1 の note 参照）。

Q を押してもアビリティが発動しない場合

BeginPlay に Give Ability（GA_Attack） があるか・ Q キーから Try Activate Ability By Class（GA_Attack） が繋がっているかを確認してください。

Play Montage and Wait の各ピン

出力ピン 発火するタイミング
On Completed モンタージュが最後のフレームまで正常に再生された
On Blend Out モンタージュが終了フェードに入った（Completed の直前）
On Interrupted 他のモンタージュや外部処理によって再生が中断された
On Cancelled アビリティがキャンセルされてモンタージュが停止した
クールダウンを追加したい場合

Duration Policy が「Has Duration」の GameplayEffect を 1 つ作り、 GA_Attack の Class Defaults で 「Cooldown Gameplay Effect Class」 にそれを設定するだけです。 設定した秒数が経過するまで TryActivate が失敗するようになります。

まとめ

Play Montage and Wait を使うと、アビリティがモンタージュの再生中に「待機」し、 アニメーションの完了・中断に合わせて自動で終了します。
Block Abilities with Tag と組み合わせることで、 攻撃モーション中の再発動をタグだけで防ぐことができます。

次のステップでは、この攻撃アビリティをさらに拡張して、 アニメーションの特定のフレームでイベントを受け取る仕組みを作ります。
