BP STEP 08

イベントで連携する

UE 5.5 版

Send Gameplay Event to Actor でイベントを送り、Wait Gameplay Event でアビリティ内から受け取る。

このステップのゴール

BP STEP 07 では「タグの変化を待つ」パターンを学びました。 このステップでは Gameplay Event という仕組みを使い、 キャラクターの Blueprint からアビリティへ直接「イベント」を送る方法を学びます。

タグが「現在の状態」を表すのに対し、Gameplay Event は「この瞬間に何かが起きた」という 一時的な通知です。数値データ（EventMagnitude）も一緒に渡せるため、 「ダメージ量を伝える」「コンボの段数を伝える」といった用途に向いています。

  • 新しいタグ Event.Attack を登録する
  • GA_EventReceiver Blueprint を作成し、Wait Gameplay Event でイベントを受け取る
  • F キーに Send Gameplay Event to Actor を設定してイベントを送る
  • F キーを押すたびに「イベント受信！」が表示されることを確認する

タグはいつでも「付いているか・外れているか」を確認できる状態管理ですが、 Gameplay Event は送った瞬間だけ発火する一方向の通知です。 Wait* パターンで受け取ることで、アビリティ側が「いつ何が起きたか」に反応できます。

Event.Attack タグを登録する

1
Project Settings で Event.Attack タグを追加する

メニューバーの Edit（編集） → Project Settings（プロジェクト設定） を開きます。 左のリストから 「GameplayTags」 を選択し、 「Manage Gameplay Tags（ゲームプレイタグを管理）」をクリックします。

「+」ボタンをクリックして新しいタグを追加します。

追加するタグ
Event.Attack

Source のドロップダウンから ini ファイルを選択して 「Add New Tag（新しいタグを追加）」 をクリックします。

既存の Ability.Attack タグとは別物

BP STEP 05 で使った Ability.Attack は「アビリティの属性タグ」です。 今回追加する Event.Attack は「イベント識別用のタグ」で、 用途が異なります。名前の前半が違うので混同しないように注意してください。

GA_EventReceiver を作成する

2
GA_EventReceiver Blueprint を作成する

Content Browser（コンテンツブラウザ）で右クリック → 「Blueprint Class（ブループリントクラス）」を選択。 親クラスの検索欄に 「GameplayAbility」 と入力して選択し、 名前を GA_EventReceiver にして作成します。

GA_EventReceiver を開くと、 Event ActivateAbility がすでに配置されています。

アビリティのグラフを組む

GA_StunWatcher（BP STEP 07）と同じ構造で、 タグの変化の代わりに Gameplay Event の受信 を待ちます。 EndAbility は呼ばず、ループで常時監視します。

3
Wait Gameplay Event でイベントを受け取る

イベントグラフの空白を右クリックして 「Commit Ability」 を配置し、 Event ActivateAbility の実行ピンに繋ぎます。

次に空白を右クリックして 「Add Custom Event...」を選択し、 イベント名を WatchForEvent にします。 Commit Ability の実行出力ピンを WatchForEvent に繋ぎます。

WatchForEvent の実行ピンから続きを作ります。 グラフの空白を右クリックして 「Wait Gameplay Event」 を検索し、ノードを配置します。 WatchForEvent の実行ピンを繋ぎます。

Wait Gameplay Event の Tag ピンに Event.Attack を設定します。

Wait Gameplay Event の 「EventReceived」出力ピンに Print String（文字列: "イベント受信！"）を繋ぎます。 Print String の実行出力ピンを WatchForEvent カスタムイベントに繋いでループを作ります。

グラフ全体の流れ
Event ActivateAbility → Commit Ability → WatchForEvent
WatchForEvent → Wait Gameplay Event
Tag: Event.Attack [EventReceived] → Print String
"イベント受信！" → WatchForEvent ↺
EndAbility は接続しない。イベントを繰り返し受け取れる。

ブループリントをコンパイルして保存してください。

F キーにイベント送信を設定する

4
キャラクターの F キーからイベントを送る

BP_ThirdPersonCharacter（使用中のキャラクター Blueprint）のイベントグラフを開きます。 グラフの空白を右クリックして 「F」 で検索し、 「F」キーの入力イベントを配置します。

F（Pressed）の実行ピンから続きを作ります。 空白を右クリックして 「Send Gameplay Event to Actor」 を検索し、ノードを配置します。

Send Gameplay Event to Actor のピン設定
Actor    : Self ノードを繋ぐ
EventTag : Event.Attack
Payload  : デフォルトのまま（空欄）
Actor ピンへの Self の繋ぎ方

グラフの空白を右クリックして 「Get a reference to self」 を検索するか、 キーボードショートカットで 「Self」 ノードを配置して Actor ピンに繋ぎます。 自分自身の ASC にイベントを送るために必要です。

F（Pressed） → Send Gameplay Event to Actor
Actor: Self / EventTag: Event.Attack

ブループリントをコンパイルして保存してください。

BeginPlay でアビリティを常時起動する

5
キャラクターの BeginPlay に追加する

BP_ThirdPersonCharacter のイベントグラフの Event BeginPlay に、 GA_EventReceiver の登録と起動を後ろに追加します。 GA_StunWatcher と同じ要領です。

追加するノード
Give Ability
  Ability Class          : GA_EventReceiver

Try Activate Ability By Class
  In Ability to Activate : GA_EventReceiver
既存のノードは残す

GA_MyFirstAbility と GA_StunWatcher の Give Ability / Try Activate Ability By Class は 削除せずそのまま残してください。 GA_EventReceiver の 2 ノードを後ろに追加します。

ブループリントをコンパイルして保存してください。

動作確認

6
イベント送受信を確認する

プレイを開始して以下の順で操作します。

  • F キーを押す → 「イベント受信！」が表示される
  • F キーを複数回押す → 押すたびに「イベント受信！」が表示される（ループ動作）
  • E キーや Q キーを押しても「イベント受信！」は表示されない（Event.Attack イベントを送っていないため）
表示されない場合

BeginPlay で GA_EventReceiver に対して Give Ability と Try Activate Ability By Class の両方が接続されているか確認してください。 また、Send Gameplay Event to Actor の Actor ピンに Self が繋がっているか確認してください。

EventMagnitude でデータを渡す（発展）

Gameplay Event のもう一つの特徴は、イベントと一緒に 数値データ（EventMagnitude） を送れることです。 タグだけでは伝えられない「どのくらいの強さか」という情報をやり取りできます。

7
EventMagnitude を送って受け取る

送る側（キャラクター Blueprint）：
Send Gameplay Event to Actor の Payload ピンを 右クリック → Split Struct Pin（構造体ピンを分割） します。 展開されたピンの中から Payload Event Magnitude に 100.0 などの数値を設定します。

受け取る側（GA_EventReceiver）：
Wait Gameplay Event の Event Data 出力ピンから ワイヤーを引き出し、「Break GameplayEventData（構造体を分割）」 ノードを配置します。 Event Magnitude ピンを Print String の文字列に繋ぎます。 （float を文字列に変換するために 「To String（Float）」 ノードを間に挟みます。）

受け取り側の変更（EventReceived ピン以降）
[EventReceived] → Break GameplayEventData
Event Data ピンから Event Magnitude → To String (Float) → Print String
"受信 Magnitude: " + 値
F キーを押すと「受信 Magnitude: 100.0」のように表示される。
Append ノードで文字列を結合する

「受信 Magnitude: 」という固定文字列と float 変換した数値を繋げるには、 「Append（文字列を追加）」 ノードを使います。 右クリックして「Append」で検索して見つかります。 ピンが足りない場合はノードを右クリックして「Add pin（ピンを追加）」で増やせます。

Wait Gameplay Event とは

Wait Gameplay Event は Ability Task の一種です。 指定したタグに一致する Gameplay Event が ASC オーナーに送られるまでアビリティが待機し、 受信した瞬間に EventReceived ピンから実行が再開されます。

ピン 説明
Tag 待ち受けるイベントのタグ。送信側と同じタグを指定する。
EventReceived（出力） 指定タグのイベントを受信した瞬間に発火する実行ピン。
Event Data（出力） 受信した FGameplayEventData 構造体。EventMagnitude・Instigator などを取り出せる。

タグとイベントの使い分け

BP STEP 07（Wait Gameplay Tag Added）と今回のイベントは、似て非なる仕組みです。

仕組み 性質 データ 向いている用途
Gameplay Tag
（Wait Tag Added / Removed）		 状態（付いているか否か） なし スタン・バフなどの「状態管理」
Gameplay Event
（Wait Gameplay Event）		 一時的な通知（起きた瞬間だけ） あり（EventMagnitude など） ダメージ・コンボ連携などの「イベント通知」

「スタン中かどうか」を確認したいなら Tag、「この瞬間に攻撃が当たった」という通知なら Event と覚えておくと選択しやすくなります。

うまくいかないときの確認ポイント

  • 「イベント受信！」が表示されない → Send Gameplay Event to Actor の Actor ピンに Self が繋がっているか
  • 「イベント受信！」が表示されない → Wait Gameplay Event の Tag ピンに Event.Attack が設定されているか
  • 「イベント受信！」が表示されない → BeginPlay で GA_EventReceiver の Give Ability と Try Activate Ability By Class が接続されているか
  • 2 回目以降のイベントで反応しない → Print String の後に WatchForEvent カスタムイベントへの接続があるか（ループになっているか）
  • EventMagnitude の値が 0 になる → Send Gameplay Event to Actor の Payload ピンを Split して Payload Event Magnitude に値を設定したか
  • すべてのブループリントをコンパイル・保存済みか
チュートリアル通りにならない場合

このチュートリアルは UE 5.5 で動作確認しています。別バージョンをお使いの場合は UE 5.5 で試してみてください。

まとめ

Send Gameplay Event to Actor + Wait Gameplay Event を使うと、 Blueprint からアビリティへ「何かが起きた」という通知をデータ付きで届けられます。
タグ（状態）・GE（時間管理）・イベント（一時通知）の 3 つを使い分けることで、 GAS らしい疎結合な設計が Blueprint だけで実現できます。

BP STEP 01〜08 を通じて、GAS の主要要素である AbilitySystemComponent・GameplayAbility・GameplayTag・GameplayEffect・Ability Tasks を Blueprint だけで動かし、Wait* パターンによるリアクティブな設計まで体験できました。
