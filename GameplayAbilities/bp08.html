BP STEP 08

イベントで連携する

UE 5.5 版

C キーを押してチャージ開始、離したときにイベントで合図。Wait Gameplay Event によるアビリティの「途中で待機」を体験する。

このステップのゴール

BP STEP 07 では GE のタグ消滅を待つ方法を学びました。 このステップでは Wait Gameplay Event を使い、 「外部から届いたイベント」を起点にアビリティの続きを動かす方法を学びます。

新しく GA_ChargeAttack というアビリティを作ります。 C キーを押すと「チャージ中...」と表示されたままアビリティが一時停止し、 C キーを離すとイベントが届いて「発射！」と表示されます。 押し続ける時間に関係なく、「離した瞬間」が発射タイミングになります。

  • Event.ChargeRelease タグを登録する
  • GA_ChargeAttack Blueprint を作成し、Wait Gameplay Event でリリースを待つ
  • C キー押下でアビリティ起動、C キーリリースで Send Gameplay Event to Actor を呼ぶ
  • 「チャージ中...」→ C を離す →「発射！」の流れを確認する

Wait Gameplay Event ノードに実行が届くと、指定タグのイベントが届くまでアビリティが一時停止します。 C キーを離したときに Send Gameplay Event to Actor でイベントを送ると、 待機中のアビリティが再開します。「いつ再開するか」をアビリティ側ではなく、 キー入力側が決められます。

Event.ChargeRelease タグを登録する

1
Event.ChargeRelease タグを追加する

メニューバーの Edit（編集） → Project Settings（プロジェクト設定） を開き、 左のリストから 「GameplayTags」 を選択します。 「Manage Gameplay Tags（ゲームプレイタグを管理）」をクリックし、 「+」 ボタンで新しいタグを追加します。

追加するタグ
Event.ChargeRelease

Source のドロップダウンから ini ファイルを選択して 「Add New Tag（新しいタグを追加）」 をクリックします。

タグの命名について

Event. で始まるタグはイベント識別用の慣習的な名前付けです。 状態を表す Status.Charging（BP STEP 07）とは異なり、 「この瞬間に何かが起きた」という一時的な通知を識別するために使います。

GA_ChargeAttack を作成する

2
GA_ChargeAttack Blueprint を作成する

Content Browser（コンテンツブラウザ）で右クリック → 「Blueprint Class（ブループリントクラス）」を選択。 親クラスの検索欄に 「GameplayAbility」 と入力して選択し、 名前を GA_ChargeAttack にして作成します。

GA_ChargeAttack を開くと、 Event ActivateAbility がすでに配置されています。

アビリティのグラフを組む

GA_ChargeAttack は「チャージ中... → イベントを待つ → 発射！ → 終了」という一直線の流れです。 ループ構造は使わず、C キーを 1 回押し離しするたびに 1 回の処理が完結します。

3
Wait Gameplay Event でリリースを待つ

イベントグラフの空白を右クリックして 「Commit Ability」 を配置し、 Event ActivateAbility の実行ピンに繋ぎます。

続けて Print String（文字列: "チャージ中..."）を繋ぎます。

続けて空白を右クリックして 「Wait Gameplay Event」 を検索してノードを配置します。 Tag ピンに Event.ChargeRelease を設定します。

Wait Gameplay Event の 「EventReceived」出力ピンに Print String（文字列: "発射！"）を繋ぎます。

最後に Print String「発射！」の実行出力ピンに 「End Ability」 を繋ぎます。

GA_ChargeAttack のグラフ全体
Event ActivateAbility → Commit Ability → Print "チャージ中..."
→ Wait Gameplay Event
Tag: Event.ChargeRelease [EventReceived] → Print "発射！" → End Ability
ループなし。C を押し離しするたびに 1 回の処理が完結する。

ブループリントをコンパイルして保存してください。

キャラクターに C キーを設定する

4
BeginPlay・Pressed・Released を設定する

BP_ThirdPersonCharacter（使用中のキャラクター Blueprint）のイベントグラフを開きます。

既存の Event BeginPlay の末尾に Give Ability（Ability Class: GA_ChargeAttack） を追加します。

次に、グラフの空白を右クリックして 「C」 で検索し、 「C」キーの入力イベントを配置します。 1 つのノードに Pressed と Released の両方の実行ピンがあります。

C（Pressed） の実行ピンに Try Activate Ability By Class（In Ability to Activate: GA_ChargeAttack） を繋ぎます。

C（Released） の実行ピンに Send Gameplay Event to Actor を繋ぎます。

Send Gameplay Event to Actor のピン設定
Actor    : Self ノードを繋ぐ
EventTag : Event.ChargeRelease
Payload  : デフォルトのまま（空欄）
C（Pressed） → Try Activate Ability By Class
GA_ChargeAttack
C（Released） → Send Gameplay Event to Actor
Actor: Self / EventTag: Event.ChargeRelease
Pressed でアビリティを起動し、Released でイベントを送って続きを動かす。
Actor ピンへの Self の繋ぎ方

グラフの空白を右クリックして 「Get a reference to self」 を検索するか、 「Self」 ノードを配置して Actor ピンに繋ぎます。 未接続のままだと ASC が見つからずイベントが届きません。

ブループリントをコンパイルして保存してください。

動作確認

5
チャージと発射を確認する

プレイを開始して以下の操作で確認します。

  • C キーを押す → 「チャージ中...」が表示される（アビリティが待機中）
  • C キーを離す → 「発射！」が表示される（イベントが届いて再開）
  • 長押しして離す → チャージ時間に関わらず、離した瞬間に「発射！」
  • C を再び押す → 同じ流れが繰り返される
「チャージ中...」が表示されない場合

BeginPlay に Give Ability（GA_ChargeAttack） があるか確認してください。 C（Pressed）から Try Activate Ability By Class（GA_ChargeAttack） が繋がっているかも確認してください。

「発射！」が表示されない場合

Send Gameplay Event to Actor の Actor ピンに Self が繋がっているか確認してください。 EventTag ピンが Event.ChargeRelease になっているかも確認してください。

EventMagnitude でデータを渡す（発展）

Gameplay Event のもう一つの特徴は、イベントと一緒に 数値データ（EventMagnitude） を送れることです。 例えば「チャージ時間に応じたダメージ倍率」を渡す用途に使えます。

6
EventMagnitude を送って受け取る

送る側（キャラクター Blueprint）：
Send Gameplay Event to Actor の Payload ピンを 右クリック → Split Struct Pin（構造体ピンを分割） します。 展開されたピンの中から Payload Event Magnitude に 100.0 などの数値を設定します。

受け取る側（GA_ChargeAttack）：
Wait Gameplay Event の Event Data 出力ピンから ワイヤーを引き出し、「Break GameplayEventData（構造体を分割）」 ノードを配置します。 Event Magnitude ピンを Print String の文字列に繋ぎます。 （float を文字列に変換するために 「To String（Float）」 ノードを間に挟みます。）

受け取り側の変更（EventReceived ピン以降）
[EventReceived] → Break GameplayEventData
Event Data ピンから Event Magnitude → To String (Float) → Print String
"発射！Magnitude: " + 値
C キーを離すと「発射！Magnitude: 100.0」のように表示される。
Append ノードで文字列を結合する

「発射！Magnitude: 」という固定文字列と変換した数値を繋げるには、 「Append（文字列を追加）」 ノードを使います。 右クリックして「Append」で検索してください。

Wait Gameplay Event とは

Wait Gameplay Event は Ability Task の一種です。 指定したタグに一致する Gameplay Event が ASC オーナーに送られるまでアビリティが待機し、 受信した瞬間に EventReceived ピンから実行が再開されます。

ピン 説明
Tag 待ち受けるイベントのタグ。送信側と同じタグを指定する。
EventReceived（出力） 指定タグのイベントを受信した瞬間に発火する実行ピン。
Event Data（出力） 受信した FGameplayEventData 構造体。EventMagnitude・Instigator などを取り出せる。

タグとイベントの使い分け

BP STEP 07（Wait Gameplay Tag Removed）と今回のイベントは、似て非なる仕組みです。

仕組み 性質 データ 向いている用途
Gameplay Tag
（Wait Tag Removed など）		 状態（付いているか否か） なし スタン・チャージなどの「状態管理」
Gameplay Event
（Wait Gameplay Event）		 一時的な通知（起きた瞬間だけ） あり（EventMagnitude など） キー離し・コンボ連携などの「瞬間通知」

「チャージ中かどうか」を管理するなら Tag、「この瞬間に離した」という通知なら Event が適切です。

うまくいかないときの確認ポイント

  • 「チャージ中...」が表示されない → BeginPlay に Give Ability（GA_ChargeAttack） があるか
  • 「チャージ中...」が表示されない → C（Pressed）から Try Activate Ability By Class（GA_ChargeAttack） が繋がっているか
  • 「発射！」が表示されない → Send Gameplay Event to Actor の Actor ピンに Self が繋がっているか
  • 「発射！」が表示されない → Send Gameplay Event to Actor の EventTag が Event.ChargeRelease になっているか
  • 「発射！」が表示されない → Wait Gameplay Event の Tag ピンが Event.ChargeRelease になっているか
  • EventMagnitude の値が 0 になる → Payload ピンを Split して Payload Event Magnitude に値を設定したか
  • すべてのブループリントをコンパイル・保存済みか
チュートリアル通りにならない場合

このチュートリアルは UE 5.5 で動作確認しています。別バージョンをお使いの場合は UE 5.5 で試してみてください。

まとめ

Wait Gameplay Event を使うと、アビリティが「外部からイベントが届くまで」一時停止できます。
キー離しのタイミングをイベントとして送るだけで、「押している間チャージ・離したら発射」が自然に実装できます。 タグ（状態）・GE（時間管理）・イベント（瞬間通知）の 3 つを使い分けることで、GAS らしい設計が Blueprint だけで実現できます。

BP STEP 01〜08 を通じて、GAS の主要要素である AbilitySystemComponent・GameplayAbility・GameplayTag・GameplayEffect・Ability Tasks を Blueprint だけで動かし、Wait* パターンによるリアクティブな設計まで体験できました。
