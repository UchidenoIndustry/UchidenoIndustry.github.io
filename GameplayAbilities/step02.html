STEP 02

最小構成のセットアップ

プロジェクトに GAS を追加して、コンパイルが通るところまで進める。

このステップのゴール

このステップでは、GAS を使い始めるために必要な最小限のセットアップを行います。 アビリティをまだ作らなくてよいです。「GAS が動く状態にする」ことだけを目標にします。

  • GAS プラグインを有効化する
  • ビルド設定に GAS を追加する
  • キャラクターに AbilitySystemComponent を追加する
  • エラーなくコンパイルが通る

C++ コードが登場します。
C++ に慣れていなくても、このステップのコードはほぼコピペで動きます。 「なぜそう書くのか」は後のステップで自然と分かってくるので、 今は動かすことを優先してください。

前提条件

GAS の機能をフル活用するには C++ プロジェクト が必要です。 ブループリントのみでもアビリティの一部は作れますが、 後のステップで必要になる AttributeSet（ステータス定義）は C++ なしでは定義できません。

新しくプロジェクトを作る場合は、プロジェクト作成画面で 「C++」を選択してください（「ブループリント」ではなく）。 テンプレートは「サードパーソン」が動作確認しやすくおすすめです。

既存のブループリントプロジェクトを使いたい場合

UE エディタのメニューから「ツール → 新規 C++ クラス」で 任意の C++ クラスを追加すると、C++ プロジェクトに変換されます。

Visual Studio のインストールからわからない場合

C++ プロジェクトをビルドするには Visual Studio の適切なインストールが必要です。 設定方法がわからない場合は、先にこちらを参照してください。
Unreal Engine 用 Visual Studio インストール設定 ― ue5study.com

セットアップ手順

1
GameplayAbilities プラグインを有効化する

UE エディタのメニューから 「編集 → プラグイン」 を開き、 検索欄に Gameplay Abilities と入力します。 見つかったら 「有効」チェックボックスをオン にして、 エディタを再起動してください。

再起動後、プロジェクトフォルダの [プロジェクト名].uproject を テキストエディタで開いて確認すると、以下のように追加されているはずです。

[プロジェクト名].uproject（確認用）
"Plugins": [
  {
    "Name": "GameplayAbilities",
    "Enabled": true
  }
]
2
Build.cs に依存モジュールを追加する

Source/[プロジェクト名]/[プロジェクト名].Build.cs を テキストエディタまたは Visual Studio / Rider で開きます。

PublicDependencyModuleNames の行を探して、 緑色の3行を追加してください。

[プロジェクト名].Build.cs
PublicDependencyModuleNames.AddRange(new string[] {
    "Core",
    "CoreUObject",
    "Engine",
    "InputCore",
    "GameplayAbilities",  // 追加
    "GameplayTags",       // 追加
    "GameplayTasks"       // 追加
});
Build.cs とは

「このプロジェクトはどのモジュール（機能群）を使うか」を Unreal Build Tool に伝えるファイルです。 GAS 関連の機能を使うには、この3つのモジュールを明示する必要があります。

3
キャラクターのヘッダーファイルを編集する

Source/[プロジェクト名]/[プロジェクト名]Character.h を開きます。 緑色の行を追加してください。

[プロジェクト名]Character.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Character.h"
#include "AbilitySystemInterface.h"  // 追加
#include "[プロジェクト名]Character.generated.h"

class UAbilitySystemComponent;  // 追加

UCLASS()
class A[プロジェクト名]Character : public ACharacter, public IAbilitySystemInterface
{
    GENERATED_BODY()

public:
    A[プロジェクト名]Character();

    // IAbilitySystemInterface の実装（GAS に必要）
    virtual UAbilitySystemComponent* GetAbilitySystemComponent() const override;

protected:
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "GAS")
    TObjectPtr<UAbilitySystemComponent> AbilitySystemComponent;

    // 既存のコードはそのまま...
};
IAbilitySystemInterface とは

「このキャラクターは GAS を持っている」と Unreal に伝えるための約束事（インターフェース）です。 これを継承することで、他のコードからこのキャラクターの ASC を取得できるようになります。

4
キャラクターの cpp ファイルを編集する

Source/[プロジェクト名]/[プロジェクト名]Character.cpp を開きます。 緑色の行を追加してください。

[プロジェクト名]Character.cpp
#include "[プロジェクト名]Character.h"
#include "AbilitySystemComponent.h"  // 追加

A[プロジェクト名]Character::A[プロジェクト名]Character()
{
    // 既存のコードはそのまま...

    // AbilitySystemComponent を作成する
    AbilitySystemComponent = CreateDefaultSubobject<UAbilitySystemComponent>(TEXT("AbilitySystemComponent"));
}

UAbilitySystemComponent* A[プロジェクト名]Character::GetAbilitySystemComponent() const
{
    return AbilitySystemComponent;
}
5
コンパイルして確認する

UE エディタのツールバーにある 「コンパイル」ボタン をクリックするか、 Visual Studio / Rider からビルドを実行します。

エラーなくコンパイルが完了したら成功です。 レベルに配置されているキャラクターを選択して 「詳細」パネルを確認すると、「GAS」カテゴリに AbilitySystemComponent が表示されているはずです。

うまくいかないときの確認ポイント

コンパイルエラーが出た場合、以下を確認してください。

  • Build.cs に3つのモジュールを追加したか（スペルミスに注意）
  • AbilitySystemInterface.h のインクルードが .h ファイルに追加されているか
  • AbilitySystemComponent.h のインクルードが .cpp ファイルに追加されているか
  • IAbilitySystemInterface の継承を追加したあと、既存のカンマが残っているか（public ACharacter, のカンマ）

このステップのまとめ

これで GAS を使う準備が整いました。
キャラクターに AbilitySystemComponent（司令塔）が追加され、 GAS の機能を呼び出せる状態になっています。

次のステップでは、実際にアビリティを作成して発動させます。 ここからはブループリントが中心になるので、C++ の作業は一段落です。
