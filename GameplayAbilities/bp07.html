BP STEP 07

複数のアビリティとタグを組み合わせる

UE 5.5 版

ダッシュ（Shift）と攻撃（Q）の相互制御を通じて、Activation Owned Tags とタグ設計パターンを体験する。

このステップのゴール

BP STEP 04 でスタン中に攻撃できなくする制御を実装しました。 このステップでは複数のアビリティを組み合わせ、 「ダッシュ中は攻撃できない」「攻撃中はダッシュできない」 という双方向の制御を実装します。

  • タグを Ability.*（アビリティの識別）と Status.*（キャラクターの状態）に分けて設計する
  • 新概念 Activation Owned Tags で、ダッシュ中に Status.Dashing タグを ASC に自動付与する
  • GA_Dash（Shift キー）を新しく作る
  • GA_Attack に設定を追加し、ダッシュ中と攻撃中の相互制御を完成させる

実際のゲームでは、アビリティの数が増えるほどタグ設計の重要性が増します。 「どのタグをどの目的で使うか」を最初に決めておくと、 後から追加するアビリティも同じルールに乗れます。 このステップで紹介する Ability.* / Status.* の分け方は、 Lyra などのサンプルプロジェクトでも採用されているパターンです。

タグを設計する

今回使うタグは3種類です。Ability.Attack は BP STEP 04 で作成済みなので、 新しく作るのは Ability.Dash と Status.Dashing の2つです。

タグ 種類 用途
Ability.Attack Ability（識別） GA_Attack を識別するラベル。攻撃中に GA_Dash をブロックするために使う。
Ability.Dash Ability（識別） GA_Dash を識別するラベル。攻撃中に GA_Dash をブロックするために参照される。
Status.Dashing Status（状態） 「ダッシュ中」というキャラクターの状態を表す。GA_Dash の発動中、ASC に自動付与される。
タグ設計の2層構造

Ability.* タグはアビリティ同士が互いを識別するために使います。Block Abilities with Tag でよく登場します。

Status.* タグはキャラクターの「いま何をしているか」を表します。Activation Blocked Tags / Activation Required Tags でよく登場します。

GA_Dash を作る

1
新しいタグを作成する

任意の GameplayAbility Blueprint の Class Defaults を開き、 AssetTags (Default AbilityTags) の「+」をクリックします。 タグブラウザで Ability.Dash を新規作成します。 同様に Status.Dashing も作成します。 Source は DefaultGameplayTags.ini（またはプロジェクト名 .ini）を選択してください。

補足

タグはどのアビリティの詳細パネルから作成しても、同じ DefaultGameplayTags.ini に保存されます。作成場所は問いません。

2
GA_Dash Blueprint を作成する

Content Browser（コンテンツブラウザ）で右クリック → Blueprint Class（ブループリントクラス）を選択します。 親クラスに GameplayAbility を指定し、 名前を GA_Dash とします。

3
GA_Dash の Class Defaults でタグを設定する

GA_Dash を開き、ツールバーの Class Defaults（クラスのデフォルト）をクリックして詳細パネルを開きます。 Tags セクションで以下の3つを設定します。

フィールド 設定値 意味
AssetTags
(Default AbilityTags)		 Ability.Dash このアビリティのラベル。他のアビリティから「ダッシュアビリティ」として参照される。
Activation Owned Tags Status.Dashing 新概念：このアビリティが発動中、ASC に Status.Dashing タグを自動付与する。EndAbility が呼ばれると自動的に除去される。
Block Abilities with Tag Ability.Dash 発動中、Ability.Dash タグを持つアビリティ（自分自身）の再発動を防ぐ。
Activation Owned Tags のポイント

Activation Owned Tags に設定したタグは、このアビリティが End Ability を呼んだ瞬間に ASC から自動的に除去されます。Remove Loose Gameplay Tags で手動削除する必要はありません。

4
GA_Dash のグラフを組む

GA_Dash のイベントグラフを開きます。 Event ActivateAbility はすでにグラフに配置されています。 Commit Ability → Delay（0.5 秒）→ End Ability の順に接続します。

Event ActivateAbility → Commit Ability → Delay
Duration: 0.5 → End Ability
Delay の間 GA_Dash が発動中なので、Activation Owned Tags で Status.Dashing が ASC に付いている。
アニメーションを使う場合

実際のゲームでは Delay の代わりに BP STEP 05 で学んだ Play Montage and Wait を使います。ここではシンプルに Delay で動作を確認します。

GA_Attack にタグを追加する

BP STEP 05 で作成した GA_Attack の Class Defaults を開き、 Tags セクションに2つの設定を追加します。

5
GA_Attack の Tags セクションを編集する
フィールド 追加する値 意味
Block Abilities with Tag Ability.Dash を追加 攻撃中は、Ability.Dash タグを持つアビリティ（GA_Dash）をブロックする。
Activation Blocked Tags Status.Dashing を追加 ASC に Status.Dashing タグがある間（ダッシュ中）、GA_Attack は発動できない。
Block Abilities with Tag と Activation Blocked Tags の使い分け

Block Abilities with Tag（攻撃 → ダッシュをブロック）: 攻撃アビリティが発動中に、指定タグ（Ability.Dash）を AssetTags に持つアビリティを直接ブロックします。「相手を名指しする」イメージ。

Activation Blocked Tags（ダッシュ中 → 攻撃をブロック）: ASC に指定タグ（Status.Dashing）が付いている間、自分自身が発動できなくなります。「状態を経由して制御する」イメージ。

2つの制御パターンの全体像

設定完了後の制御の仕組みをまとめます。

制御したい状況 使ったフィールド 仕組み
ダッシュ中は攻撃できない GA_Dash: Activation Owned Tags: Status.Dashing
GA_Attack: Activation Blocked Tags: Status.Dashing		 GA_Dash の発動中、ASC に Status.Dashing が付く。GA_Attack はこのタグがある間は発動できない。
攻撃中はダッシュできない GA_Attack: Block Abilities with Tag: Ability.Dash GA_Attack の発動中、Ability.Dash を AssetTags に持つアビリティ（GA_Dash）がブロックされる。
ダッシュ中にダッシュできない GA_Dash: Block Abilities with Tag: Ability.Dash GA_Dash の発動中、自分自身の再発動がブロックされる。

キャラクター BP にダッシュを組み込む

6
BeginPlay に Give Ability を追加する

キャラクター Blueprint を開き、Event BeginPlay のグラフで 既存の Give Ability (GA_Attack) の次に Give Ability (GA_Dash) を追加します。

Event BeginPlay → Give Ability
GA_Attack → Give Ability
GA_Dash
GA_Dash を Give しないと TryActivateAbilityByClass を呼んでも何も起きない。
7
Shift キーで GA_Dash を発動させる

イベントグラフの空白部分を右クリックし、 「Left Shift」を検索して Keyboard Events（キーボードイベント） → Left Shift ノードを配置します。 Pressed 出力から Try Activate Ability By Class に繋ぎ、 In Ability to Activate に GA_Dash を指定します。

Left Shift
Pressed → Try Activate Ability By Class
In Ability to Activate: GA_Dash
補足

サードパーソンテンプレートのデフォルト設定では Left Shift には何も割り当てられていません。別のアクションと競合する場合はキーを変更してください。

動作確認

Play（プレイ）してキャラクターを動かし、以下を確認します。

  • Q キーを押すと攻撃が発動する
  • Shift キーを押すと 0.5 秒ほどダッシュ（Delay）が発動する
  • Shift 押下中（Delay の間）に Q キーを押しても攻撃が発動しない
  • Q キーの攻撃中に Shift キーを押してもダッシュが発動しない
  • Q や Shift を素早く連打しても二重発動しない
動作確認のヒント

ブロックが機能しているか確認するには、Try Activate Ability By Class の Return Value（戻り値）を Print String に繋ぎます。ブロックされたときは false が表示されます。

まとめ

このステップでは、2つのアビリティが互いを制御し合う仕組みを実装しました。

  • Activation Owned Tags：アビリティが発動中、指定タグを ASC に自動付与する。EndAbility で自動除去されるため、手動管理が不要。
  • Block Abilities with Tag：発動中に、特定タグを AssetTags に持つ他のアビリティをブロックする（相手を名指し）。
  • Activation Blocked Tags：ASC に特定タグがある間、そのアビリティ自身が発動できなくなる（状態を経由して制御）。

この3つのフィールドを組み合わせると、スタン・無敵・チャージ中など あらゆる「状態と制御の関係」をタグだけで表現できます。 次のステップでは、このタグ設計を活かしてコンボ攻撃（3連撃）を実装します。
