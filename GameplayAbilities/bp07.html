BP STEP 07

タグの変化を待つ

UE 5.5 版

Wait Gameplay Tag Removed でタグ消滅を待ち、「構え → 解放」の 2 段階で処理するアビリティを作る。

このステップのゴール

これまで作ったアビリティは「発動 → 処理 → 終了」という一方向の流れでした。 このステップでは Wait* ノード（Ability Tasks） を使い、 アビリティがグラフの途中で一時停止し、条件が揃ったら続きを動かせるようにします。

新しく GA_Charge というアビリティを作ります。 R キーを押すと「構え中...」と表示され、2 秒後に自動で「解放！」と表示されます。 2 秒間の待機は Tick やタイマーではなく、 Wait Gameplay Tag Removed が GE のタグ消滅を待つことで実現します。

  • Status.Charging タグと GE_Charge（Has Duration 2s）を用意する
  • GA_Charge Blueprint を作成し、GE を付与したあと Wait Gameplay Tag Removed でタグ消滅を待つ
  • R キーを押すと「構え中...」→ 2 秒後 →「解放！」と順番に表示されることを確認する

Wait Gameplay Tag Removed ノードに実行が届くと、Status.Charging タグが消えるまでアビリティが一時停止します。 GE_Charge の Duration（2 秒）が終わってタグが消えると、自動的に続きが動き出します。 Tick もタイマーも不要で、「GE が切れるまで待つ」処理がノード 1 つで完結します。

Status.Charging タグを登録する

1
Status.Charging タグを追加する

メニューバーの Edit（編集） → Project Settings（プロジェクト設定） を開き、 左のリストから 「GameplayTags」 を選択します。 「Manage Gameplay Tags（ゲームプレイタグを管理）」をクリックし、 「+」 ボタンで新しいタグを追加します。

追加するタグ
Status.Charging

Source のドロップダウンから ini ファイルを選択して 「Add New Tag（新しいタグを追加）」 をクリックします。

Status.Stunned がある場合

BP STEP 04 以降で Status.Stunned を追加済みであれば、 同じ Status 親タグ配下に Status.Charging が追加されます。

GE_Charge を作成する

2
GE_Charge Blueprint を作成する

Content Browser（コンテンツブラウザ）で右クリック → 「Blueprint Class（ブループリントクラス）」を選択。 親クラスの検索欄に 「GameplayEffect」 と入力して選択し、 名前を GE_Charge にして作成します。

GE_Charge を開き、Class Defaults（クラスのデフォルト）の詳細パネルで以下を設定します。

GE_Charge の設定
Duration Policy    : Has Duration
Duration Magnitude : 2.0

次に 「Gameplay Effect」セクション → 「Components」の「+」 をクリックし、 ドロップダウンから 「Grant Tags to Target Actor」 を選択します。

追加されたコンポーネント（表示名: 「Target Tags (Granted to Actor)」）の 「Add Tags」→「Add to Inherited」 に Status.Charging を追加します。

ブループリントをコンパイルして保存してください。

GE_Stun と同じ作り方

BP STEP 06 の GE_Stun と同じ手順です。 Duration を 2s に、付与するタグを Status.Charging に変えるだけです。

GA_Charge を作成する

3
GA_Charge Blueprint を作成する

Content Browser（コンテンツブラウザ）で右クリック → 「Blueprint Class（ブループリントクラス）」を選択。 親クラスの検索欄に 「GameplayAbility」 と入力して選択し、 名前を GA_Charge にして作成します。

GA_Charge を開くと、イベントグラフに Event ActivateAbility がすでに配置されています。

アビリティのグラフを組む

GA_Charge は「GE を付与 → タグが消えるまで待機 → 終了」という一直線の流れです。 ループ構造は使わず、1 回の発動で完結します。

4
GE を付与して Wait Tag Removed に繋げる

イベントグラフの空白を右クリックして 「Commit Ability」 を配置し、 Event ActivateAbility の実行ピンに繋ぎます。

続けて空白を右クリックして 「Apply Gameplay Effect to Owner」 を検索してノードを配置します。 Gameplay Effect Class に GE_Charge を設定します。 Commit Ability の実行出力ピンに繋ぎます。

続けて Print String（文字列: "構え中..."）を配置して繋ぎます。

続けて 「Wait Gameplay Tag Removed」 を検索してノードを配置します。 Tag ピンに Status.Charging を設定します。

Wait Gameplay Tag Removed の 「Removed」出力ピンに Print String（文字列: "解放！"）を繋ぎます。

最後に Print String「解放！」の実行出力ピンに 「End Ability」 を繋ぎます。

GA_Charge のグラフ全体
Event ActivateAbility → Commit Ability → Apply GE to Owner
GE_Charge
→ Print "構え中..." → Wait Tag Removed
Status.Charging
[Removed] → Print "解放！" → End Ability
ループなし。1 回の発動で「構え中...」→ 2 秒 →「解放！」→ 終了。
「Apply Gameplay Effect to Owner」が見つからない場合

GA Blueprint 内で使えるノードです。見つからない場合は代わりに、 My Blueprint → Components → AbilitySystem をドラッグして参照ノードを作り、 そこから 「ApplyGameplayEffectToSelf」 を繋ぐ方法（BP STEP 06 と同じ手順）でも同じ結果になります。

ブループリントをコンパイルして保存してください。

キャラクターに割り当てる

5
BeginPlay と R キーに設定する

BP_ThirdPersonCharacter（使用中のキャラクター Blueprint）のイベントグラフを開きます。

既存の Event BeginPlay の末尾に Give Ability（Ability Class: GA_Charge） を追加します。Try Activate Ability By Class は不要です。

次に、グラフの空白を右クリックして 「R」 で検索し、 「R」キーの入力イベントを配置します。 R（Pressed）の実行ピンに Try Activate Ability By Class（In Ability to Activate: GA_Charge） を繋ぎます。

R（Pressed） → Try Activate Ability By Class
GA_Charge
R を押すたびに 1 回の「構え → 解放」サイクルが実行される。
BeginPlay では TryActivate しない理由

GA_Charge は R キーを押すたびに起動し、1 回の処理で完結します。 常時動作させる必要がないため、BeginPlay では登録（Give Ability）だけ行います。

ブループリントをコンパイルして保存してください。

動作確認

6
2 段階実行を確認する

プレイを開始して以下の操作で確認します。

  • R キーを押す → 「構え中...」が即座に表示される
  • 2 秒後 → 「解放！」が自動で表示される
  • R キーをもう一度押す → 同じ流れが繰り返される
「構え中...」が表示されない場合

BeginPlay に Give Ability（GA_Charge） があるか確認してください。 R（Pressed）から Try Activate Ability By Class（GA_Charge） が繋がっているかも確認してください。

「解放！」が表示されない場合

GE_Charge の「Target Tags (Granted to Actor)」→「Add to Inherited」に Status.Charging が設定されているか確認してください。 Wait Gameplay Tag Removed の Tag ピンに Status.Charging が設定されているかも確認してください。

Wait* ノードとは

Wait Gameplay Tag Removed は Ability Task という仕組みで実装された非同期ノードです。 アビリティが「中断して条件を待ち、条件が満たされたら続きを実行する」ことを可能にします。

ノード 一時停止する条件 再開するタイミング
Wait Gameplay Tag Removed 指定タグが ASC に付いている間 指定タグが ASC から削除された瞬間
Wait Gameplay Tag Added 指定タグが ASC に付いていない間 指定タグが ASC に追加された瞬間

GA_Charge のグラフが「構え中... → ここで 2 秒止まる → 解放！」と一直線に読める点に注目してください。 Tick や条件分岐がなくても、タグの変化がアビリティの「続き」を自動で動かします。

Ability Task とは

Ability Task は GameplayAbility 内で非同期処理を行うための仕組みです。 Wait Gameplay Tag Removed 以外にも Wait Input Pressed（入力を待つ）・Wait Gameplay Event（イベントを待つ）など多くの種類があります。 グラフで「Wait」と検索すると一覧が表示されます。次のステップではイベントを待つ Wait Gameplay Event を使います。

うまくいかないときの確認ポイント

  • 「構え中...」が表示されない → BeginPlay に Give Ability（GA_Charge） があるか
  • 「構え中...」が表示されない → R（Pressed）から Try Activate Ability By Class（GA_Charge） が繋がっているか
  • 「構え中...」が表示されない → GA_Charge のグラフで Apply Gameplay Effect to Owner に GE_Charge が設定されているか
  • 「解放！」が表示されない → GE_Charge の Grant Tags to Target Actor に Status.Charging が設定されているか
  • 「解放！」が表示されない → Wait Gameplay Tag Removed の Tag ピンに Status.Charging が設定されているか
  • すべてのブループリントをコンパイル・保存済みか
チュートリアル通りにならない場合

このチュートリアルは UE 5.5 で動作確認しています。別バージョンをお使いの場合は UE 5.5 で試してみてください。

まとめ

Wait Gameplay Tag Removed を使うと、アビリティがグラフの途中で一時停止し、 タグが消えた瞬間に自動で続きを動かせます。
Tick もタイマーも不要。GE の Duration という「既存の仕組み」をそのままアビリティの待機時間として使えるのが、GAS らしい設計です。
