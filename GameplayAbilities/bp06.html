BP STEP 06

攻撃フレームを制御する

UE 5.5 版

AnimNotify Blueprint から Send Gameplay Event を呼び、Wait Gameplay Event でアビリティ内に受け取る。

このステップのゴール

BP STEP 05 の GA_Attack は、モンタージュが終わると自動で終了しましたが、 「アニメーションのどのフレームで何が起きるか」は制御できていませんでした。 このステップでは AnimNotify Blueprint を使って 攻撃の当たり判定フレームを指定し、 Wait Gameplay Event でアビリティ内に受け取ります。

  • AnimNotify Blueprint（AN_AttackHit）を作成して、特定フレームで Gameplay Event を送る
  • AM_Attack に Notify を配置して「当たり判定フレーム」を設定する
  • GA_Attack に Wait Gameplay Event を追加して、イベントを受け取った瞬間に処理する
  • Q を押すと攻撃アニメーションが再生され、指定フレームで「ヒット！」が表示されることを確認する

Lyra や ActionRPG などのサンプルプロジェクトで「アニメーションの特定フレームで何かを起こす」場合、 この AnimNotify → Send Gameplay Event → Wait Gameplay Event というパターンが多用されます。 タイミングをアニメーション側が持つことで、コード側はイベントを待つだけでよくなります。

GA_Attack 内で Play Montage and Wait と Wait Gameplay Event が並行して動く。

Event.Attack.Hit タグを登録する

1
Event.Attack.Hit タグを作成する

AnimNotify から GA_Attack に送るイベントの識別に使うタグです。 GA_Attack の Class Defaults を開き、Tags セクションの AssetTags の「+」をクリックします。 タグブラウザで Event.Attack.Hit を新規作成します。

AssetTags に追加する必要はありません。 タグをシステムに登録することが目的です。 タグブラウザで作成したあとはキャンセルして閉じてください。

Project Settings から登録する方法

Edit（編集）→ Project Settings（プロジェクト設定）→ GameplayTags → 「Manage Gameplay Tags（Gameplay タグを管理）」でも同じ登録ができます。

AnimNotify Blueprint を作成する

2
AN_AttackHit Blueprint を作成する

Content Browser で右クリック → Blueprint Class（ブループリントクラス） を選択。 親クラスの検索欄に 「AnimNotify」 と入力して選択し、 名前を AN_AttackHit にして作成します。

AN_AttackHit を開き、左側の My Blueprint パネルで FUNCTIONS → Override（+ボタン）→ Received Notify を選択します。 イベントグラフに Received Notify ノード（Mesh Component と Animation ピンを持つ）が配置されます。

次のように接続します。

AN_AttackHit のグラフ
Received Notify
Mesh Component, Animation
Mesh Component → Get Owner → Send Gameplay Event
to Actor
Event Tag: Event.Attack.Hit
Get Owner の戻り値（Actor）を Send Gameplay Event to Actor の「Actor」ピンに繋ぐ。
Received Notify の実行ピンも Send Gameplay Event to Actor の実行ピンに繋ぐ。
Send Gameplay Event to Actor の Payload について

Payload ピン（FGameplayEventData 型）は空のままでも動作します。 ヒット時のダメージ量などを渡したい場合は Make GameplayEventData ノードで組み立てて接続します。

ブループリントをコンパイルして保存してください。

AM_Attack に Notify を配置する

3
攻撃フレームに AN_AttackHit を配置する

AM_Attack を開きます。 タイムライン上の 「Notifies」トラック（「Notify」行）で、 攻撃が当たるタイミング（モンタージュの中盤あたり）を 右クリック → Add Notify（通知を追加） → AN_AttackHit を選択して Notify を配置します。

AN_AttackHit が選択肢に表示されない場合

Notify のドロップダウンには「New Notify...（新規通知を作成...）」と既存の AnimNotify クラスが並びます。 AN_AttackHit が表示されない場合は、一度 Content Browser に戻って AN_AttackHit を開いてコンパイル・保存してから再試行してください。

モンタージュを保存してください。

GA_Attack のグラフを更新する

4
Sequence と Wait Gameplay Event を追加する

GA_Attack を開き、グラフを変更します。 Commit Ability と Play Montage and Wait の間に Sequence ノードを挿入します。

GA_Attack グラフの変更前 → 変更後
【変更前】
Event ActivateAbility → Commit Ability → Play Montage and Wait → ...

【変更後】
Event ActivateAbility → Commit Ability → Sequence
                                          ├─ Then 0 → Play Montage and Wait
                                          │           [On Completed]   → End Ability
                                          │           [On Blend Out]   → End Ability
                                          │           [On Interrupted] → End Ability
                                          │           [On Cancelled]   → End Ability
                                          └─ Then 1 → Wait Gameplay Event
                                                      (Event Tag: Event.Attack.Hit)
                                                      [Event Received] → Print "ヒット！"

Sequence ノードは、空白を右クリック → 「Sequence」で検索して配置できます。 Then 0 と Then 1 に、それぞれ Play Montage and Wait と Wait Gameplay Event を繋ぎます。

GA_Attack の更新後グラフ
Event ActivateAbility → Commit Ability → Sequence
[Then 0] → Play Montage and Wait
Montage to Play: AM_Attack
[On Completed] → End Ability
[On Blend Out] → End Ability
[On Interrupted] → End Ability
[Then 1] → Wait Gameplay Event
Event Tag: Event.Attack.Hit
[Event Received] → Print "ヒット！"
Sequence は Then 0（Play Montage and Wait）を起動し、続けて Then 1（Wait Gameplay Event）も起動する。
両タスクが同時に走り、AnimNotify が発火するとイベントが GA_Attack に届く。
Sequence と Ability Task の動作について

Play Montage and Wait などの Ability Task ノードは、Blueprint の通常の待機（Delay など）とは異なり、 ノードの入力が実行されると即座にタスクを登録して次の処理へ進みます。 そのため Sequence の Then 0 が Play Montage and Wait を起動したあと、 Then 1 の Wait Gameplay Event も続けて起動され、両者が並行して動作します。

Wait Gameplay Event のパラメーター

Event Tag: Event.Attack.Hit を設定します。
Only Trigger Once: チェック ON（true）を推奨します。 1 回の攻撃で複数の AnimNotify が発火するモンタージュを使う場合は OFF のままにすると、 ヒットごとに EventReceived が発火します。

ブループリントをコンパイルして保存してください。

動作確認

5
攻撃フレームでヒットが検出されることを確認する
  • Q キーを押す → 攻撃アニメーションが再生される
  • アニメーションが Notify フレームに到達する → Output Log に「ヒット！」が表示される
  • 「ヒット！」の表示タイミングが Notify を配置した攻撃フレームと一致している
「ヒット！」が表示されない場合の確認点

AN_AttackHit の Get Owner の戻り値が Send Gameplay Event to Actor の Actor ピンに繋がっているか確認してください。 Actor ピンが未接続だと ASC が見つからず、イベントが送信されません。

AM_Attack に Notify（AN_AttackHit）が配置されているか確認してください。 Montage エディターのタイムラインで確認できます。

GA_Attack の Wait Gameplay Event の Event Tag が Event.Attack.Hit になっているか確認してください。

このパターンが優れている理由

アニメーションの「当たり判定フレーム」はキャラクターが腕を振り切る瞬間など、 アーティストが決める要素です。 Notify をアニメーション側に配置することで、 コード側はタイミングを知らなくてよくなります。

比較 タイマーや Delay でタイミングを指定する AnimNotify → Wait Gameplay Event パターン
タイミングの変更 コードの数値を調整して再コンパイル Montage エディターで Notify を動かすだけ
アニメーション速度変更 ズレが生じる（固定秒数なので） 自動で追従する（アニメーションの割合で決まるため）
役割の分離 プログラマーがタイミングも管理 アーティストが Notify でタイミングを管理

まとめ

AN_AttackHit（AnimNotify Blueprint）が攻撃フレームで Send Gameplay Event を呼び、 GA_Attack の Wait Gameplay Event がそれを受け取る。 この「AnimNotify → Send Gameplay Event → Wait Gameplay Event」パターンは、 GAS を使うサンプルプロジェクトで広く使われている設計です。

Blueprint 編では ASC、GameplayAbility、GameplayTag の 3 要素を中心に学んできました。 ダメージの数値管理（AttributeSet）や複雑なゲームプレイ効果（GameplayEffect）を 本格的に扱うには C++ が必要になります。 GAS をさらに深く使いたい場合は C++ 編でステップアップしてください。
