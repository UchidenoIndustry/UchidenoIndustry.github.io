BP STEP 06

Duration GE でスタン自動解除

UE 5.5 版

Apply Gameplay Effect to Actor で GE_Stun を付与し、Status.Stunned タグを 3 秒間自動管理する。

このステップのゴール

BP STEP 04 では E キーを押している間だけスタン状態にする手動管理を実装しました。 このステップでは、同じスタンを「E キーを一度押したら 3 秒間持続して自動解除」 という GameplayEffect 方式に置き換えます。

  • GE_Stun（Has Duration / Status.Stunned タグ付与）を作成する
  • Apply Gameplay Effect to Actor ノードで GE_Stun を付与する
  • E キー 1 回でスタンが 3 秒間継続し、自動で解除されることを確認する

GE_Stun を作成する

BP STEP 05 で作った GE_AttackCooldown と構造は同じです。 付与するタグだけが異なります。

1
GE_Stun を作成する

Content Browser（コンテンツブラウザ）で右クリック → 「Blueprint Class（ブループリントクラス）」を選択。 親クラスの検索欄に 「GameplayEffect」 と入力して選択し、 名前を GE_Stun にして作成します。

GE_Stun を開き、 「Class Defaults（クラスのデフォルト）」を選択。 詳細パネルの 「Duration」セクションで以下を設定します。

Duration セクション
Duration Policy          : Has Duration
Scalable Float Magnitude : 3.0

次に 「Gameplay Effect」セクションの Components の 「+」をクリックします。 ドロップダウンから 「Grant Tags to Target Actor」を選択します。

「Target Tags (Granted to Actor)」コンポーネントが追加されます。 「Add Tags」の中の 「Add to Inherited」の欄に Status.Stunned を設定します。 （BP STEP 04 で作成済みのタグなので、新規作成は不要です。）

GE_Stun の最終設定
Duration Policy          : Has Duration
Scalable Float Magnitude : 3.0
Grant Tags to Target Actor
  └ Add to Inherited     : Status.Stunned
GE_AttackCooldown との違い

GE_AttackCooldown は Cooldown.Attack タグを付与していました。 GE_Stun は Status.Stunned タグを付与します。 Duration・設定手順はまったく同じです。

ブループリントをコンパイルして保存してください。

スタン実装を GE 方式に置き換える

BP STEP 04 で組んだキャラクターの Blueprint を修正します。 E キーの Pressed・Released に繋いだノードを GE 方式に変更します。

2
E キーのスタン実装を置き換える

BP_ThirdPersonCharacter（使用中のキャラクター Blueprint）のイベントグラフを開きます。 BP STEP 04 で組んだ以下のノードを探してください。

現在（BP STEP 04 の実装）
E（Pressed） → Add Loose Gameplay Tags
Status.Stunned → Print String
"Stunned!"
E（Released） → Remove Loose Gameplay Tags
Status.Stunned → Print String
"Stun Removed"

次の手順で変更します。

① E（Released）から繋がるノードをすべて削除します。
Remove Loose Gameplay Tags と続く Print String を選択して Delete キーで削除してください。 E ノードの Released ピンから出ているワイヤーも切断されます。

② E（Pressed）から繋がる Add Loose Gameplay Tags を削除します。
Print String「Stunned!」はそのまま残しておきます。

③ My Blueprint パネルの Components から AbilitySystem をグラフにドラッグします。
作成された参照ノードからワイヤーを引き出し、「apply gameplay effect」 で検索して ApplyGameplayEffectToSelf (AbilitySystem) を選択します。 E（Pressed）→ ApplyGameplayEffectToSelf → Print String「Stunned! (3s)」の順に接続します。

変更後
E（Pressed） → ApplyGameplayEffectToSelf
Target: AbilitySystem
GE Class: GE_Stun → Print String
"Stunned! (3s)"
E（Released）のノードは不要。GE が 3 秒後に自動解除する。
ApplyGameplayEffectToSelf のピン設定

Target ピンには My Blueprint パネルの Components から AbilitySystem をドラッグした参照ノードを繋いでください。
Gameplay Effect Class ピンに GE_Stun を選択します。
Level は 1.0 のままで問題ありません。

ブループリントをコンパイルして保存してください。

動作確認

3
スタン自動解除を確認する

プレイを開始して以下の順で操作します。

  • E キーを一度押す → 「Stunned! (3s)」が表示される
  • すぐに E キーを離す → スタンは継続する（以前は離すと解除されていた）
  • 3 秒間は Q キーを押しても何も表示されない（Activation Blocked Tags が機能中）
  • 3 秒後に Q キーを押す → 「Ability Activated!」が表示される（自動解除済み）
E を連打した場合

3 秒のクールダウン中に再び E を押すと GE_Stun が再付与されます。 デフォルトでは先に付与された GE と同時に存在し、それぞれ独立してカウントします。 今回のチュートリアルでは、この動作はそのままで構いません。

ApplyGameplayEffectToSelf とは

ApplyGameplayEffectToSelf は AbilitySystem コンポーネントのメソッドで、 ASC を持つキャラクター自身に GameplayEffect を付与します。 My Blueprint パネルの AbilitySystem コンポーネントから呼び出すため、 Has Matching Gameplay Tag と同じ要領で使えます。

ピン 説明
Target AbilitySystem コンポーネントの参照。My Blueprint パネルからドラッグして接続。
Gameplay Effect Class 付与する GE のクラス。今回は GE_Stun。
Level GE の強度。Scalable Curve を使う場合に影響。通常は 1.0 のまま。
戻り値（Active GE Handle） 付与した GE の識別子。Infinite GE を手動解除するときに使う。Has Duration GE では不要。
他のキャラクターに付与したい場合

今回は自分自身に付与しましたが、他のキャラクターに付与する場合は ApplyGameplayEffectToTarget (AbilitySystem) を使います。 例えば「攻撃がヒットした相手に GE_Stun を付与する」実装に使えます。

Loose Gameplay Tags と GE によるタグ付与の違い

BP STEP 04 の手動管理と今回の GE 方式を比べると、用途の違いが見えてきます。

方法 付与 解除 向いている用途
Add Loose Gameplay Tags ノードで直接付与 Remove Loose Gameplay Tags で手動解除 テスト・単純な状態フラグ
GE + Grant Tags to Target Actor Apply Gameplay Effect to Actor Has Duration なら時間切れで自動解除 時間制限のあるデバフ・バフ

Loose Gameplay Tags は「押している間だけ」のようなリアルタイム制御に向いています。 GE 方式は「付与した瞬間から n 秒間」という時間管理が自動化されるため、 より実際のゲームの挙動に近くなります。

うまくいかないときの確認ポイント

  • スタンが発動しない → ApplyGameplayEffectToSelf の Target に AbilitySystem コンポーネントの参照が繋がっているか
  • スタンが発動しない → GE_Stun の Components に Grant Tags to Target Actor が追加されているか
  • スタンが発動しない → Grant Tags to Target Actor の Add to Inherited に Status.Stunned が設定されているか
  • スタンは発動するが 3 秒で解除されない → Duration Policy が Has Duration になっているか
  • スタン中も Q キーが発動する → GA_MyFirstAbility の Activation Blocked Tags に Status.Stunned が設定されているか（BP STEP 04 で設定済みのはず）
  • すべてのブループリントをコンパイル・保存済みか
チュートリアル通りにならない場合

このチュートリアルは UE 5.5 で動作確認しています。別バージョンをお使いの場合は UE 5.5 で試してみてください。

まとめ

GameplayEffect の Has Duration と Grant Tags to Target Actor を組み合わせると、 「タグの付与・時間管理・自動解除」をコード不要で一括管理できます。
Loose Gameplay Tags が手動のフラグ管理なら、 GE 方式はそれを時間管理付きに高度化したものです。

Blueprint 編の6ステップを通じて、GAS の主要要素 （AbilitySystemComponent・GameplayAbility・GameplayTag・GameplayEffect） を Blueprint だけで動かすことができました。
