BP EX 02

ヒット検出・ダメージ・GameplayCue

UE 5.5 版

攻撃 GA にヒット検出・ダメージ・GameplayCue を追加する。C++ なしで「当たったら HP が減り、エフェクトが出る」を実装する。

このページのゴール

  • GA 内で Sphere Trace を実行し、攻撃フレームのタイミングで相手キャラクターを検出する。
  • UE 標準の Apply Damage で相手の HP float 変数を減らす。
  • GameplayCueNotify_Static Blueprint でヒットエフェクト（パーティクル・サウンド）を再生する。
  • ヒット検出・ダメージ・エフェクトのすべてを Blueprint のみで実装する。
前提

BP STEP 06 の GA_Attack（Play Montage and Wait + AnimNotify + Wait Gameplay Event のパターン）が完成していることが前提です。

UE 標準ダメージ vs GAS ネイティブ

UE のダメージ管理には 2 種類のアプローチがあります。 このページでは UE 標準（Apply Damage / Event AnyDamage） を採用し、HP を相手の float 変数で管理します。 Blueprint のみで完結する代わりに、GAS の Modifier やクールダウン計算とは連動しません。

GAS ネイティブなダメージ管理（GameplayEffect の Modifier で AttributeSet.HP を変化させる）には C++ が必要です。 このページ末尾の「次のステップへ」セクションで C++ 編 STEP 04〜05 を案内します。

STEP 1 — 相手キャラクターを準備する

1
HP 変数を追加する

攻撃を受ける相手の Character Blueprint（例: BP_Enemy）を開きます。

My Blueprint パネル → VARIABLES の「+」から変数を追加します。

変数名型デフォルト値
HPFloat100.0

変数を追加したら Compile（コンパイル） して保存します。

2
Event AnyDamage を設定する

相手 Character Blueprint のイベントグラフで右クリック → 「Event AnyDamage」 を検索して配置します。

以下のノードを接続します。

Event AnyDamage → HP = HP - Damage → Clamp（Min: 0, Max: 100） → Set HP
Event AnyDamage の「Damage」ピンを減算に、結果を Clamp してから HP 変数にセットする。
HP の上限を変数にする場合

MaxHP 変数を別途追加して Clamp の Max ピンに繋ぐと、キャラクターごとに上限を変えられます。このチュートリアルでは固定値 100.0 を使います。

ダメージを Print String で確認する

Set HP のあとに Print String（HP） を繋いでおくと、画面にリアルタイムで HP 値が表示されて動作確認が楽になります。確認が終わったら削除してください。

STEP 2 — GameplayCue アセットを作る

3
GC_AttackHit を作成する

Content Browser（コンテンツブラウザ）で右クリック → Blueprint Class（Blueprintクラス）。 親クラスの検索窓に GameplayCueNotify_Static と入力して選択し、名前を GC_AttackHit とします。

作成した GC_AttackHit を開き、 Class Defaults（クラスデフォルト）の詳細パネルで Gameplay Cue Tag フィールドをクリックします。 タグブラウザが開くので、新規タグ GameplayCue.Attack.Hit を作成して設定します。

GameplayCue タグの命名規則

GameplayCue で使うタグは必ず GameplayCue. プレフィックスが必要です。このプレフィックスがないと GAS が GameplayCue として認識しません。また GameplayCue タグは通常の Gameplay Tag と同じシステムに登録されます。

GameplayCueNotify_Static とは

一回だけ発火するエフェクト（ヒットフラッシュ・サウンドなど）に使います。持続するエフェクト（炎上・氷結など）には GameplayCueNotify_Actor を使いますが、このページでは Static を使います。

4
On Execute を組む

GC_AttackHit のイベントグラフを開き、 My Blueprint パネル → FUNCTIONS → Override（+）→ On Execute を選択します。 グラフに On Execute イベントノードが追加されます。

On Execute ノードは MyTarget（Actor）と Parameters（GameplayCueParameters）の 2 つの出力ピンを持ちます。 Parameters から位置情報を取り出し、エフェクトを再生します。

On Execute（MyTarget, Parameters） → Break GameplayCueParameters
Parameters ピンを右ドラッグ → Break
Location → Spawn System at Location
Niagara System: お好みのエフェクト
Location → Play Sound at Location
Sound: お好みの SE
Break GameplayCueParameters → Location フィールドがヒット位置（ImpactPoint）になる。Spawn と Play Sound はどちらも On Execute の実行ピンから繋ぐ。
パーティクルとサウンドのアセットについて

このチュートリアルでは使用するアセットを指定しません。エンジンコンテンツ（Starter Content など）に含まれる既存の Niagara System やサウンドアセットを使うと手軽に確認できます。UE 5.5 では Spawn System at Location（Niagara）が標準ですが、Cascade の Spawn Emitter at Location も同様に使えます。

Normal を使った回転

Break GameplayCueParameters の Normal フィールドにはヒット面の法線方向が入ります。Make Rot from X で回転に変換し、Spawn のパラメータに渡すと、エフェクトがヒット面の法線方向に向いて発生します。

STEP 3 — GA_Attack を改修する

bp06 で作った GA_Attack のグラフには Wait Gameplay Event（Event.Attack.Hit） の Event Received 出力がすでにあります。 この出力に Sphere Trace → Apply Damage → Execute GameplayCue の処理を追加します。

BP STEP 08 まで進んでいる場合

BP STEP 08 のコンボ構成では Q キーが GA_Combo1 に割り当てられています。このページの STEP 3 の処理は GA_Attack の代わりに GA_Combo1（および各コンボ段の GA）の Wait Gameplay Event（Event.Attack.Hit） に追加してください。

5
ヒット検出・ダメージ・GameplayCue を追加する

GA_Attack を開きます。 Wait Gameplay Event（Event.Attack.Hit） の Event Received 実行ピンから以下のグラフを組みます。

Apply Damage と Execute GameplayCue は同じ True 実行ピンから順に繋ぐ（Apply Damage → Execute GameplayCue）。Return Value が false（空振り）のときは Branch の False パスに進むため、無効発火はない。

各ノードの配置手順

Get Avatar Actor from Actor Info

グラフ空白を右クリック → 「Get Avatar Actor from Actor Info」で検索・配置。GA のオーナーキャラクターへの参照が得られます。

Sphere Trace For Objects

グラフ空白を右クリック → 「Sphere Trace For Objects」で検索・配置。Object Types ピンの右の「+」を押して要素を追加し、ドロップダウンから Pawn を選択します。Actors to Ignore は Make Array ノードで Avatar Actor を 1 つ渡します。

Make Gameplay Cue Parameters from Hit Result

グラフ空白を右クリック → 「Make Gameplay Cue Parameters from Hit Result」で検索・配置。HitResult ピンに Sphere Trace の Out Hit（HitResult 型）を繋ぎます。これによりヒットポイントの位置・法線が Parameters に格納されます。

Execute GameplayCue On Actor (Burst)

グラフ空白を右クリック → 「Execute GameplayCue On Actor (Burst)」で検索・配置。Actor ピンに Avatar Actor を、Gameplay Cue Tag に GameplayCue.Attack.Hit を設定します。

Event Instigator について

Apply Damage の Event Instigator ピン（Controller 型）は省略可能です。ダメージ元のプレイヤーを識別したい場合は、Avatar Actor から Cast to Pawn → Get Controller で取得して繋いでください。

Only Trigger Once の設定

Wait Gameplay Event（Event.Attack.Hit） の Only Trigger Once を true にすると、1 回発火後にタスクが自動終了します。攻撃アニメーション 1 回につき 1 ヒットの設計では true を推奨します。同一アニメーション中に複数ヒットさせる場合は false のままにしてください。

動作確認

Play（プレイ）して以下のシナリオを確認します。

  • ヒット検出：攻撃を発動すると、AN_AttackHit が発火するタイミングで相手キャラクターが検出される。Sphere Trace の Draw Debug Type を For Duration にすると範囲が可視化できる。
  • ダメージ：相手の HP 変数が 25 減算される（Print String などで確認）。
  • GameplayCue：ヒット位置にパーティクルとサウンドが再生される。
  • ミス（空振り）：相手がいない状態で攻撃してもエフェクトが発生しない（Return Value が false のため Branch の True パスが実行されない）。
Sphere Trace のデバッグ表示

Sphere Trace For Objects の Draw Debug Type を For Duration にすると、ビューポートにトレース範囲が赤いワイヤーフレームで表示されます。ヒット判定の範囲調整に活用し、本番前に None に戻してください。

手や剣の位置からトレースしたい場合

現在の実装はキャラクターの Actor Location（足元中心）を起点にしています。 実際のゲームでは手先や剣の刃先からトレースしたいことがほとんどです。 その場合は Skeleton に Socket を作成し、そのワールド座標を Start／End に使います。

Socket の作り方： Skeleton エディタを開き（キャラクターのメッシュをダブルクリック → Skeleton ツールを選択）、 対象のボーン（例: hand_r）を右クリック → Add Socket。 剣の刃元と刃先に 2 つ作ると、その間をトレースできます。

GA 内での取得： Get Avatar Actor from Actor Info → Cast to Character → Get Mesh → Get Socket Location（Socket Name: hand_r） で Socket のワールド座標が得られます。これを Sphere Trace の Start に使い、End を刃先の Socket 座標にすれば剣のスウィング範囲をトレースできます。

剣が別の Actor としてアタッチされている場合は、武器 Actor の参照を取得して同様に Get Mesh → Get Socket Location を呼びます。

まとめ

  • Sphere Trace For Objects：AN_AttackHit の発火タイミングに合わせてキャラクター前方のエリアを検索。Actors to Ignore で自己攻撃を防ぎ、Object Types: Pawn で敵のみを検出する。
  • Apply Damage / Event AnyDamage：UE 標準のダメージパイプライン。GA 側で Apply Damage を呼ぶだけで、相手の Event AnyDamage が発火して HP を減算できる。C++ 不要。
  • Make Gameplay Cue Parameters from Hit Result / Execute GameplayCue On Actor (Burst)：ヒット結果の位置・法線を Parameters に変換し、GC_AttackHit に渡す。GameplayCueNotify_Static にエフェクトをカプセル化しておくことで、GA 側は Cue タグ 1 つを渡すだけでよい。

この「GA が Sphere Trace → Apply Damage → Execute GameplayCue」のパターンは、 コンボ攻撃の各段・スキル攻撃・範囲攻撃など幅広い攻撃アビリティに応用できます。
