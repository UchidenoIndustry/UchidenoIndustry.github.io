BP STEP 02

Blueprint で始める

UE 5.5 対応

C++ を使わず、Blueprint だけで GAS を動作させる最小構成のセットアップ。

このステップのゴール

  • GAS プラグインを有効化する
  • キャラクターの Blueprint に AbilitySystemComponent を追加する
  • AbilitySystemInterface を Blueprint で実装する
  • Get Ability System Component ノードが機能することを確認する

このステップは Blueprint のみで完結します。
C++ のコードを書く必要はありません。Build.cs の変更も不要です。

GAS プラグインを有効化する

1
Gameplay Abilities プラグインを有効にする

UE エディタのメニューから 「編集 → プラグイン」 を開きます。 検索欄に 「Gameplay Abilities」 と入力してプラグインを探し、 チェックボックスをオンにしてください。

確認ダイアログが出たら「今すぐ再起動」を押してエディタを再起動します。 再起動が完了したらプロジェクトを再度開いてください。

Build.cs の変更は不要

C++ コードを書かない Blueprint 専用のセットアップでは、 Build.cs に GAS モジュールを追加する必要はありません。 プラグインを有効化するだけで使えます。

AbilitySystemComponent を追加する

2
キャラクターの Blueprint を開く

コンテンツブラウザからキャラクターのブループリントを開きます。 サードパーソンテンプレートを使用している場合は Content/ThirdPerson/Blueprints/BP_ThirdPersonCharacter です。

3
AbilitySystemComponent コンポーネントを追加する

ブループリントエディタ左側の 「コンポーネント」パネルで 「+ 追加」ボタンをクリックします。 検索欄に 「Ability System Component」 と入力して 候補を選択してください。

コンポーネントが追加されたら、名前はデフォルトの AbilitySystemComponent のままにしてください。この名前をあとで参照します。

ブループリントをコンパイルして保存してください。

AbilitySystemInterface を実装する

コンポーネントを追加しただけでは、GAS のノード（Get Ability System Component）が 「このキャラクターの ASC はどれか」を認識できません。 AbilitySystemInterface を実装することで、 GAS が正しく ASC を見つけられるようになります。

4
クラスの設定でインターフェースを追加する

ブループリントエディタ上部のツールバーにある 「クラスの設定」ボタンをクリックします。 詳細パネルの「インターフェース」セクションで 「+ 追加」をクリックし、 検索欄に 「AbilitySystem」 と入力してください。

「AbilitySystemInterface」を選択して追加します。 「My Blueprint」パネルの「インターフェース」セクションに GetAbilitySystemComponent という関数が表示されます。

5
GetAbilitySystemComponent を実装する

「My Blueprint」パネルの「インターフェース」セクションに表示された GetAbilitySystemComponent をダブルクリックして関数グラフを開きます。

「My Blueprint」パネルの 「コンポーネント」セクションから AbilitySystemComponent を関数グラフにドラッグします。 作成された Getter ノードの出力ピンを Return ノードの戻り値に接続してください。

GetAbilitySystemComponent 関数の中身
Get AbilitySystemComponent → Return Node
Return Value（接続する）

ブループリントをコンパイルして保存してください。

なぜこの実装が必要か

GAS の多くのノードは内部で IAbilitySystemInterface::GetAbilitySystemComponent() を呼び出して ASC を探します。この関数を実装しておくことで、 キャラクターのイベントグラフや他のシステムから Get Ability System Component ノードが正しく機能するようになります。

動作確認

6
ASC が取得できることを確認する

キャラクターのイベントグラフに戻り、 Event BeginPlay から以下のようにノードをつないでください。

Event BeginPlay → Get Ability System Component
Target: Self → Is Valid → Branch
└ True → Print String
"GAS 準備完了！"
└ False → Print String
"エラー: ASC が見つかりません"

ブループリントをコンパイルして保存し、プレイを開始してください。 画面左上に 「GAS 準備完了！」と表示されれば成功です。

確認が終わったら

この Print String ノードは動作確認用です。 次のステップに進む前に削除するか、そのまま残しておいても構いません。

うまくいかないときの確認ポイント

  • 「Gameplay Abilities」プラグインが有効化され、エディタが再起動されているか
  • コンポーネントパネルに AbilitySystemComponent が表示されているか
  • クラスの設定 → インターフェースに AbilitySystemInterface が追加されているか
  • GetAbilitySystemComponent 関数内で AbilitySystemComponent を Return しているか
  • ブループリントをコンパイルして保存してからプレイしているか

このステップのまとめ

C++ を書かずに GAS を使える状態になりました。
「コンポーネントを追加 → インターフェースを実装」 の2ステップが Blueprint での GAS セットアップのポイントです。
